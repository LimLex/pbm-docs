# Percona Backup for MongoDB 0.5.0

Percona 很高兴于 2019 年 6 月 17 日宣布我们最新软件产品 Percona Backup for MongoDB 0.5.0 的早期版本。Percona Backup for MongoDB 的 GA 版本计划于 2019 年晚些时候发布。

Percona Backup for MongoDB 是一个分布式、低影响的解决方案，用于创建 MongoDB 分片集群和副本集的一致备份。这是一个用于在 MongoDB 分片集群（或单个副本集）中创建一致备份并将这些备份恢复到特定时间点的工具。Percona Backup for MongoDB 使用分布式客户端/服务器架构来执行备份/恢复操作。

该项目受到（并打算取代）[Percona-Lab/mongodb_consistent_backup 工具](https://github.com/Percona-Lab/mongodb_consistent_backup) 的启发。

Percona Backup for MongoDB 支持版本 3.6 或更高版本的 Percona Server for MongoDB 或 MongoDB Community Server，并启用了 MongoDB 复制。支持的平台的二进制文件以及包含源代码的 tarball 可从 [Percona Backup for MongoDB 下载页面](https://www.percona.com/downloads/percona-backup-mongodb/LATEST/) 获得。有关 Percona Backup for MongoDB 和安装步骤的更多信息，请参阅[文档](https://www.percona.com/doc/percona-backup-mongodb)。

Percona Backup for MongoDB 0.5.0 具有以下特点：

* 支持在 Amazon Simple Storage Service 存储上存储备份元数据。

* Percona Backup for MongoDB 的 API 引入了 HTTP 基本身份验证，以防止未经授权的用户在访问 API 端口时运行备份或恢复数据。

* 为了优化网络资源的使用，不再需要在 `mongos` 上使用 pbm-agent，备份协调器会自动建立与适当的 `mongos` 实例的连接。

* `pbmctl list nodes` 的输出现在包括副本集名称并通知节点的备份状态。

Percona 不建议将此版本用于生产环境，因为其 API 和配置字段仍可能发生变化。它仅具有基本的 API 级别安全性。请在[我们的错误跟踪系统](https://jira.percona.com) 中报告您遇到的任何错误。

## 新功能和改进


* [93](https://jira.percona.com/browse/93)：支持在 AWS S3 上存储备份元数据。


* [99](https://jira.percona.com/browse/99)：**pbm-agent** 在 `mongos` 上已弃用。


* [105](https://jira.percona.com/browse/105)：如果使用主节点类型进行备份，则记录警告


* [122](https://jira.percona.com/browse/122)：将副本集名称包含到 `pmbctl list nodes` 的输出中


* [130](https://jira.percona.com/browse/130)：为 gRPC 服务器（API 和 RPC）添加 HTTP 基本身份验证


* [139](https://jira.percona.com/browse/139)：支持在 `pmbctl list nodes` 的输出中列出备份状态


* [170](https://jira.percona.com/browse/170)：启用在 `mongorestore` 中设置 'stopOnError' 属性，以确保正在恢复的数据的一致性。
