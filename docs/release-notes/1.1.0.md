# Percona Backup for MongoDB 1.1.0

Percona 很高兴于 2020 年 1 月 16 日宣布发布 Percona Backup for MongoDB 1.1.0。

Percona Backup for MongoDB 是一个分布式、低影响的解决方案，用于创建 MongoDB 分片集群和副本集的一致备份。这是一个用于在 MongoDB 分片集群（或单个副本集）中创建一致备份并将这些备份恢复到特定时间点的工具。该项目受到（并打算取代）Percona-Lab/mongodb_consistent_backup 工具的启发。

Percona Backup for MongoDB 支持 [Percona Server for MongoDB](https://www.percona.com/software/mongo-database/percona-server-for-mongodb) 或版本 3.6 或更高版本的 [MongoDB Community Server](https://www.mongodb.com/download-center/community)，并启用了 [MongoDB 复制](https://docs.mongodb.com/manual/replication/)。支持的平台的二进制文件以及包含源代码的 tarball 可从 [Percona Backup for MongoDB 下载页面](https://www.percona.com/downloads/percona-backup-mongodb/) 获得。有关 Percona Backup for MongoDB 和安装步骤的更多信息，请参阅文档。

Percona Backup for MongoDB 1.1.0 引入了新的 `pbm config` 命令，除了配置文件之外，还可以从命令行配置存储。此命令有效地取代了只能从配置文件读取存储配置的 pbm store。

```
$ pbm config --set storage.s3.bucket="operator-testing"
```

## 新功能

* [PBM-344](https://jira.percona.com/browse/PBM-344)：新的 pbm config 命令，支持从命令行配置存储。

## 改进

* [PBM-361](https://jira.percona.com/browse/PBM-361)：改进了使用 oplog 时时间戳的处理。

## 已修复的错误

* [PBM-214](https://jira.percona.com/browse/PBM-214)：如果主节点在恢复操作期间变得不可用，`pbm-agent` 可能会崩溃，恢复命令永远运行。

* [PBM-279](https://jira.percona.com/browse/PBM-279)：`pbm-agent` 可能使用无效的配置文件启动。

* [PBM-338](https://jira.percona.com/browse/PBM-338)：失败的备份可能出现在 pbm list 命令的输出中。

* [PBM-362](https://jira.percona.com/browse/PBM-362)：如果没有健康的从节点，从主节点调用 `pbm backup` 可能会失败。

* [PBM-369](https://jira.percona.com/browse/PBM-369)：当在集群中使用 TLS 时，副本集无法建立连接。
