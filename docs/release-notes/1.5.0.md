# Percona Backup for MongoDB 1.5.0

| 发布日期 | 2021 年 5 月 10 日   |
|------------- | ---------------|
| 安装 | [安装 Percona Backup for MongoDB](../installation.md) |


Percona Backup for MongoDB 是一个分布式、低影响的解决方案，用于创建 MongoDB 分片集群和副本集的一致备份。这是一个用于在 MongoDB 分片集群（或单个副本集）中创建一致备份并将这些备份恢复到特定时间点的工具。

## 新功能

* [PBM-596](https://jira.percona.com/browse/PBM-596)：Azure Blob Storage 支持

* [PBM-488](https://jira.percona.com/browse/PBM-488)：创建权重或标记方法以影响哪个 `pbm-agent` 节点将进行备份

## 改进

* [PBM-662](https://jira.percona.com/browse/PBM-662)：基于 `admin.pbmLock` 而不是配置设置显示 PITR 状态

* [PBM-494](https://jira.percona.com/browse/PBM-494)：在自动选择中优先选择（健康的）隐藏从节点而不是任何其他节点

## 已修复的错误

* [PBM-642](https://jira.percona.com/browse/PBM-642)：在 `pbm status` 输出的代理列表中显示 `priority=0` 成员

* [PBM-636](https://jira.percona.com/browse/PBM-636)：恢复后不同的集合 UUID（感谢 Nikolay 报告此问题以及 Dmitry Kuzmin 的贡献）

* [PBM-646](https://jira.percona.com/browse/PBM-646)：在备份期间停止平衡器，以确保它在恢复期间不会开始运行

* [PBM-635](https://jira.percona.com/browse/PBM-635)：在启动备份之前等待领导者的元数据

* [PBM-490](https://jira.percona.com/browse/PBM-490)：使用集群时间作为快照开始时间
