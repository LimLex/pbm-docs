# Percona Backup for MongoDB 2.0.0 (2022-09-21)

| 发布日期 | 2022 年 9 月 21 日  |
|------------- | ---------------|
| 安装 | [安装 Percona Backup for MongoDB](../installation.md) |

Percona Backup for MongoDB 是一个分布式、低影响的解决方案，用于创建 MongoDB 分片集群和副本集的一致备份。这是一个用于在 MongoDB 分片集群（或非分片副本集）中创建一致备份并将这些备份恢复到特定时间点的工具。

## 发布亮点

* 物理备份和恢复现已正式可用。这使您可以在生产环境中使用它们。
* [静态数据加密](../features/physical.md#physical-restores-with-data-at-rest-encryption) 支持物理备份和恢复。这使您能够遵守数据安全法规，并节省处理大型数据集的时间。
* 通过[跟踪物理恢复进度](../usage/restore-progress.md)，您可以清楚地了解恢复操作，并及时响应任何更改或问题。
* [逻辑备份和恢复](../features/selective-backup.md) 现在可以选择性地进行。这是技术预览功能 [^1]，但它使您能够仅处理所需的数据子集，从而节省数据库维护时间和存储成本。



## 新功能

* [PBM-790](https://jira.percona.com/browse/PBM-790)：添加进行选择性备份和恢复的能力。这是技术预览功能 [^1]

## 改进

- [PBM-896](https://jira.percona.com/browse/PBM-896)：启用静态数据加密的物理恢复
- [PBM-898](https://jira.percona.com/browse/PBM-898),[PBM-811](https://jira.percona.com/browse/PBM-811), [PBM-812](https://jira.percona.com/browse/PBM-812)：在物理恢复期间记录临时 mongod 进程
- [PBM-911](https://jira.percona.com/browse/PBM-911)：重命名 `pbm status` 和 `pbm list` 输出中备份的完成时间
- [PBM-813](https://jira.percona.com/browse/PBM-813)：在物理恢复期间启用 `pbm-agents` 通过存储发送心跳


## 已修复的错误

- [PBM-909](https://jira.percona.com/browse/PBM-909)：在物理恢复期间考虑存储设置
- [PBM-943](https://jira.percona.com/browse/PBM-943)：修复文件系统存储的 pbm status 显示
- [PBM-929](https://jira.percona.com/browse/PBM-929)：在 oplog 应用期间重新创建时间序列集合的 UUID



[^1]: 技术预览功能尚未准备好供企业使用，也不包含在 SLA 支持中。它们包含在此版本中，以便用户可以在未来 GA 版本中完整发布功能（或如果认为该功能无用则删除该功能）之前提供反馈。此功能可能会从技术预览到 GA 发生变化（API、CLI 等）。
