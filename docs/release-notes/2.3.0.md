# Percona Backup for MongoDB 2.3.0 (2023-09-25)

[Installation](../installation.md){.md-button}


Percona Backup for MongoDB 是一个分布式、低影响的解决方案，用于创建 MongoDB 分片集群和副本集的一致备份。这是一个用于在 MongoDB 分片集群（或非分片副本集）中创建一致备份并将这些备份恢复到特定时间点的工具。

## 发布亮点

* 对 MongoDB 4.2 的支持已弃用。Percona Backup for MongoDB 中的现有功能仍与 MongoDB 4.2 和 Percona Server for MongoDB 4.2 兼容；但是，不再针对此版本测试进一步的增强和错误修复。
* [查看备份内容](../usage/describe-backup.md) 的能力改进了在经常创建和/或删除数据库的环境中备份的故障排除。
* [在混合部署中进行物理备份](../features/physical.md#physical-backups-in-mixed-deployments) 的能力，使用 MongoDB Community 和 Percona Server for MongoDB (PSMDB) 节点，简化了仍在评估或迁移其数据集以使用 PSMDB 的组织的备份流程。
* 改进了时间点恢复的基础备份快照验证，使逻辑和物理备份的恢复流程保持一致。

### 支持的平台更改

* Percona Backup for MongoDB 的 Docker 镜像现在可用于 ARM64 架构。这改善了使用基于 ARM 的 MacBook 的开发人员的 PBM 用户体验。
* Percona Backup for MongoDB 软件包现在可用于 Debian 12 (bookworm)。Percona Server for MongoDB 正在通过 7.0.x 版本添加对 Debian 12 的支持。

## 改进

* [PBM-1147](https://jira.percona.com/browse/PBM-1147), [PBM-885](https://jira.percona.com/browse/PBM-885) - 支持符号链接作为文件系统存储类型的 `path`，并在重新同步期间正确检索数据
* [PBM-1162](https://jira.percona.com/browse/PBM-1162), [PBM-1166](https://jira.percona.com/browse/PBM-1166) - 改进物理和增量备份作为时间点恢复基础的验证
* [PBM-1171](https://jira.percona.com/browse/PBM-1171) - 添加在混合部署中进行物理备份的能力
