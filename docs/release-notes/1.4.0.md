# Percona Backup for MongoDB 1.4.0

| 发布日期 | 2020 年 12 月 24 日  |
|------------- | ---------------|
| 安装 | [安装 Percona Backup for MongoDB](../installation.md) |


Percona Backup for MongoDB 是一个分布式、低影响的解决方案，用于创建 MongoDB 分片集群和副本集的一致备份。这是一个用于在 MongoDB 分片集群（或单个副本集）中创建一致备份并将这些备份恢复到特定时间点的工具。

## 新功能

* [PBM-345](https://jira.percona.com/browse/PBM-345)：集中日志

* [PBM-435](https://jira.percona.com/browse/PBM-435)：`pbm status` 命令

## 改进

* [PBM-572](https://jira.percona.com/browse/PBM-572)：将 'pbm list' 等中的备份 'name' 更改为一致的时间（~= 结束时间）而不是开始时间

* [PBM-556](https://jira.percona.com/browse/PBM-556)：引入操作 ID

## 已修复的错误

* [PBM-595](https://jira.percona.com/browse/PBM-595)：具有不同 rset 名称的分片备份

* [PBM-604](https://jira.percona.com/browse/PBM-604)：'pbm list' 命令的压缩标志不会更改输出

* [PBM-602](https://jira.percona.com/browse/PBM-602)：如果 PBM 由于范围不足而无法上传 oplog 块，则在存储上创建空的 PITR 文件

* [PBM-597](https://jira.percona.com/browse/PBM-597)：在启用 PITR 切片时正确处理 mongo 故障
