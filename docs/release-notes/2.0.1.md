# Percona Backup for MongoDB 2.0.1 (2022-10-12)

| 发布日期 | 2022 年 10 月 12 日  |
|------------- | ---------------|
| 安装 | [安装 Percona Backup for MongoDB](../installation.md) |


Percona Backup for MongoDB 是一个分布式、低影响的解决方案，用于创建 MongoDB 分片集群和副本集的一致备份。这是一个用于在 MongoDB 分片集群（或非分片副本集）中创建一致备份并将这些备份恢复到特定时间点的工具。

## 发布亮点

* 支持[使用客户提供的、在客户侧管理的密钥进行服务器端加密 (SSE-C)](../details/s3-storage.md#server-side-encryption)，使您能够使用您选择的 S3 兼容存储，从而防止供应商锁定并节省 AWS KMS（密钥管理服务）的成本。
* [远程配置 Percona Backup for MongoDB](../manage/configure-remotely.md) 的能力简化了在 Docker、Kubernetes 或其他云服务中部署 PBM 时的管理。
* 为 Percona Backup for MongoDB 配置 sidecar 模式的能力改善了其作为 [Kubernetes Operator for MongoDB](https://docs.percona.com/percona-operator-for-mongodb/index.html) 一部分的操作。
* 故障排除增强：
  
    - [定义日志时区并动态跟踪日志](../reference/pbm-commands.md#pbm-logs) 的能力。
    - 在 `pbm status` 输出中将仲裁节点指示为不支持的节点



## 新功能

- [PBM-733](https://jira.percona.com/browse/PBM-733)：允许更改 PBM 中日志的时区
- [PBM-877](https://jira.percona.com/browse/PBM-877)：使 PBM 能够从 stdin 读取配置文件

## 改进

- [PBM-949](https://jira.percona.com/browse/PBM-949)：改进对仲裁节点的处理
- [PBM-948](https://jira.percona.com/browse/PBM-948)：对齐 `pbm describe-backup` 和 `pbm describe-restore` 输出
- [PBM-941](https://jira.percona.com/browse/PBM-941)：为 PBM 添加 sidecar 模式，以防止在容器中错误配置 `pbm-agent` 时数据库崩溃
- [PBM-922](https://jira.percona.com/browse/PBM-922)：为 S3 提供商添加 SSE-C 支持（感谢 Richard Bateman 报告此问题并为其做出贡献）
- [PBM-897](https://jira.percona.com/browse/PBM-897)：添加动态跟踪 PBM 日志的能力

- [PBM-966](https://jira.percona.com/browse/PBM-966)：修复了 `pbm-agents` 对不存在的时间戳进行时间点恢复的错误处理
