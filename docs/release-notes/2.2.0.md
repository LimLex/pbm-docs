# Percona Backup for MongoDB 2.2.0 (2023-07-13)

| 发布日期 | 2023 年 7 月 13 日  |
|------------- | ---------------|
| 安装 | [安装 Percona Backup for MongoDB](../installation.md) |


Percona Backup for MongoDB 是一个分布式、低影响的解决方案，用于创建 MongoDB 分片集群和副本集的一致备份。这是一个用于在 MongoDB 分片集群（或非分片副本集）中创建一致备份并将这些备份恢复到特定时间点的工具。

## 发布亮点

* [从物理备份进行时间点恢复](../usage/pitr-physical.md) 现在已自动化，类似于从逻辑备份进行时间点恢复。这减轻了您的 DBA 在物理恢复之上执行手动 oplog 重放的负担，确保数据一致性并统一了 PBM 的用户体验。
* 大型数据集的所有者现在可以使用 PBM [创建外部物理备份](../features/snapshots.md) 作为 EBS 快照或通过他们选择的技术，并从这些备份中恢复，PBM 保证数据一致性。因此，他们受益于提高的性能和减少的停机时间，并确保他们的数据保持一致。这是技术预览功能。
* [从物理和增量备份恢复到新环境](../features/restore-new-env.md) 的能力，使用不同的副本集名称，扩展了物理恢复的兼容环境集。

## 改进

* [PBM-1055](https://jira.percona.com/browse/PBM-1055) - 添加配置物理和增量恢复的副本集重映射的能力
* [PBM-1118](https://jira.percona.com/browse/PBM-1118) - 支持使用副本集重映射的恢复的自定义分片名称
* [PBM-1132](https://jira.percona.com/browse/PBM-1132) - 禁止对物理和增量备份及恢复使用 `--ns` 标志
* [PBM-1133](https://jira.percona.com/browse/PBM-1133) - 为 s3 存储桶添加虚拟托管样式 URL 支持
* [PBM-1062](https://jira.percona.com/browse/PBM-1062) - 添加每种备份类型的受支持 PSMDB 版本检查
