# Percona Backup for MongoDB 2.9.0 ({{date.2_9_0}})

[Installation](../installation.md){.md-button}


Percona Backup for MongoDB 是一个分布式、低影响的解决方案，用于创建 MongoDB 分片集群和副本集的一致备份，并将这些备份恢复到特定时间点。

## 发布亮点

此版本提供以下功能和改进：

### Amazon Linux 2023 支持

Percona Backup for MongoDB 现在在 Amazon Linux 2023 (AL23) 上可用并完全支持，简化了其 AWS 部署。您可以安全地在 AL23 上运行 PBM，以构建安全、稳定、高性能的环境来开发和运行云应用程序，并与各种 AWS 服务和开发工具无缝集成。 

### `pbm-agent` 设置的集中管理

您现在可以在单个文件中定义 `pbm-agent` 配置并使用它启动代理。配置包括 MongoDB 连接 URI、自定义日志路径和备份的并行工作线程数。此外，您可以在运行时调整日志级别，而无需重启 `pbm-agent`。

这有助于您将所有 `pbm-agent` 设置保存在一个地方并简化其管理。 

了解有关如何[使用配置文件启动 pbm-agent](../manage/start-agent-with-config.md) 以及[配置自定义日志路径的所有方法](../manage/logpath.md) 的更多信息

### 改进 Docker 镜像的安全性

Percona Backup for MongoDB Docker 镜像现在基于 Universal Base Image (UBI) 版本 9，其中包括最新的安全修复。这使镜像符合 Red Hat 认证，并确保容器在 Red Hat OpenShift Container Platform 上无缝工作。
